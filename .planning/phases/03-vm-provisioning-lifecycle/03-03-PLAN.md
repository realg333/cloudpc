  <action>
Create teardown flow:

1. **src/lib/provisioning/teardown.ts** — processExpiredVms():
   - Find ProvisionedVm where status IN ('vm_ready','expiring') AND expiresAt < now, limit 10
   - For each: transition to 'destroying'
   - Call vultr.deleteInstance(vultrInstanceId)
   - On success: set destroyedAt = now, status = 'destroyed'
   - Update Order: set status = 'completed' where orderId
   - On error: set failureCode, failureMessage, lastProviderResponse, status = 'failed'

2. **src/app/api/cron/process-teardown/route.ts** — GET handler:
   - Verify CRON_SECRET
   - Call processExpiredVms()
   - Return 200 { destroyed: N }

3. Optional: transition vm_ready → expiring when within N minutes of expiry (e.g. 5 min). Can be done in same process or separate. For MVP, go directly from vm_ready to destroying when expired.
  </action>

Implement reconciliation and cost safety:

1. **src/lib/provisioning/cost-safety.ts**:
   - **isProvisioningAllowed()**: Check PROVISIONING_ENABLED (default true). If false, return false (kill switch).
   - **getActiveVmCount()**: Count ProvisionedVm where status IN ('provisioning','vm_ready','expiring','destroying').
   - **canProvision()**: Return isProvisioningAllowed() && getActiveVmCount() < (VULTR_MAX_ACTIVE_VMS or 50).
   - Update src/lib/provisioning/processor.ts: before processing job, call canProvision(); if false, skip and set nextRetryAt.

2. **src/lib/vultr/client.ts** — add listInstances(): GET https://api.vultr.com/v2/instances, returns instances array. Used by reconciliation.

3. **src/lib/provisioning/reconciliation.ts** — runReconciliation():
   - List VMs in DB with vultrInstanceId and status IN (vm_ready, provisioning, destroying)
   - For each: call vultr.getInstance(id). If 404: VM gone in Vultr but we think it exists — mark status=failed, failureCode='RECONCILIATION_MISSING'
   - Call vultr.listInstances(), filter by label prefix 'cloudpc-'
   - For each Vultr instance: find ProvisionedVm by vultrInstanceId. If not found: orphan — log for admin (or optionally destroy)
   - Find ProvisionedVm with status vm_ready and expiresAt < now: should have been torn down — call processExpiredVms logic for these
   - Return summary: { checked, missing, orphans, expiredFixed }

4. **src/app/api/cron/reconciliation/route.ts** — GET handler:
   - Verify CRON_SECRET
   - Call runReconciliation()
   - Return 200 with summary

5. **hasActiveOrder**: Current impl (status='paid') already excludes 'completed'. When order.status='completed' after teardown, hasActiveOrder returns false. No change needed; verify in tests.
