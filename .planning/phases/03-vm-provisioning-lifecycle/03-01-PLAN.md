Extend prisma/schema.prisma:

1. **MachineProfile** — add: vultrPlanId String?, vultrRegion String? (nullable for non-Vultr profiles). Add @@index on vultrPlanId if needed.

2. **ProvisioningJob** — new model:
   - id (cuid)
   - orderId String @unique (idempotency: one job per order)
   - status String @default("pending") — pending | processing | completed | failed
   - createdAt, updatedAt
   - order Order @relation
   - provisionedVm ProvisionedVm? (one-to-one via ProvisionedVm.provisioningJobId)

3. **ProvisionedVm** — new model:
   - id (cuid)
   - orderId String @unique (one VM per order)
   - provisioningJobId String? @unique
   - status String — payment_confirmed | provisioning_requested | provisioning | vm_ready | expiring | destroying | destroyed | failed
   - vultrInstanceId String? @unique
   - machineProfileId String
   - machineProfile MachineProfile @relation
   - order Order @relation
   - provisioningJob ProvisioningJob? @relation
   - provisioningStartedAt DateTime?
   - readyAt DateTime?
   - expiresAt DateTime?
   - destroyedAt DateTime?
   - failureCode String?
   - failureMessage String?
   - lastProviderResponse Json?
   - retryCount Int @default(0)
   - nextRetryAt DateTime?
   - connectionMethod String? (e.g. "parsec")
   - connectionState String?
   - connectionMetadata Json?
   - windowsUsername String?
   - secretReference String?
   - ipAddress String?
   - label String?
   - hostname String?
   - createdAt, updatedAt
   - @@index([status]), @@index([vultrInstanceId]), @@index([expiresAt])

4. **Order** — add status value "completed" (when VM destroyed). Existing: pending_payment, paid, canceled. Add completed for post-destroy.

5. **ProvisioningConfig** (or env/config) — for cost safety: add a simple model or document in CONTEXT that maxActiveVms Int, provisioningEnabled Boolean (kill switch) live in env vars: VULTR_MAX_ACTIVE_VMS, PROVISIONING_ENABLED. Do NOT add a new table for MVP; use process.env with defaults. Document in a config module.

6. Add Order relation: provisionedVm ProvisionedVm?, provisioningJob ProvisioningJob?
