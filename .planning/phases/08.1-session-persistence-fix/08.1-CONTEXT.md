## Phase Boundary

Sessão de autenticação persiste corretamente; usuário não é redirecionado para login indevidamente. O app usa cookie JWT + banco (Prisma/Session). Causas prováveis identificadas: URL base incorreta em produção, SESSION_SECRET, cookie secure.
</domain>

<decisions>
## Implementation Decisions

### Causa raiz principal
- **getBaseUrl em redirects** — Logout, 2FA verify, signup e verify-email usam `request.url` para redirect. Em Vercel/proxy, `request.url` pode ser URL interna. O login já usa `getBaseUrl(request)`; padronizar em todos os fluxos.

### Arquivos a alterar
- `src/app/(auth)/logout/route.ts`
- `src/app/api/auth/two-factor/verify/route.ts`
- `src/app/api/auth/signup/route.ts`
- `src/app/(auth)/verify-email/route.ts`
- `src/app/api/auth/resend-verification/route.ts`
- `src/app/api/auth/two-factor/disable/route.ts`
- `src/app/api/auth/two-factor/setup/confirm/route.ts`

### getBaseUrl
- Extrair para `src/lib/auth/getBaseUrl.ts` ou adicionar em `session.ts`; usar `x-forwarded-host` e `x-forwarded-proto` quando presentes.

### SESSION_SECRET
- Documentar em .env.example; verificar que está definido em produção (checklist).

### Opcional (se persistir)
- `prefetch={false}` em links críticos (dashboard, orders, profile)
- Logs temporários em getSessionFromCookies para debug
</decisions>
