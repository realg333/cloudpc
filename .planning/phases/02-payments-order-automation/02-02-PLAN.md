Create src/app/api/webhooks/payments/route.ts:

1. Use raw body for signature verification. Next.js 15 App Router: request.text() for raw body. If route uses bodyParser, disable it for this route — Next.js does not parse by default for route handlers; use const rawBody = await request.text().

2. Get gateway from createMockGateway() (or future getGateway()). Call verifyWebhookSignature(rawBody, Object.fromEntries(request.headers.entries())). If false, return NextResponse.json({ error: 'Invalid signature' }, { status: 401 }).

3. Parse: const payload = gateway.parseWebhookPayload(JSON.parse(rawBody)). If null, return NextResponse.json({ error: 'Invalid payload' }, { status: 400 }).

4. Idempotency: try { await prisma.paymentLog.create({ data: { gatewayEventId: payload.eventId, eventType: payload.eventType, orderId: payload.orderId, outcome: payload.status, metadata: { event: payload.eventType } } }); } catch (e) { if (e.code === 'P2002') return NextResponse.json({ received: true }, { status: 200 }); throw e; }

5. Load order: const order = await prisma.order.findUnique({ where: { id: payload.orderId }, include: { user: true } }). If !order or order.status !== 'pending_payment', return NextResponse.json({ received: true }, { status: 200 }) (ack to stop retries).

6. PLAN-03: If payload.status === 'paid', call hasActiveOrder(order.userId). If true, user already has another paid order — do NOT mark this order paid. Instead: await prisma.order.update({ where: { id: order.id }, data: { status: 'canceled' } }); return NextResponse.json({ received: true }, { status: 200 }). (hasActiveOrder returns true for any paid order; we have not yet updated this order, so the paid order must be a different one.)

7. State transitions:
   - payload.status === 'paid': update order status to 'paid'; create Payment record (orderId, gatewayPaymentId from payload or generate, amountCents: order.amountCents, currency: order.currency, method: 'pix'|'crypto' from metadata, status: 'completed')
   - payload.status === 'failed' or 'expired': update order status to 'canceled'

8. Log: include orderId, userId, amount, method, eventType, outcome in PaymentLog (already stored). Add console.log or structured log for traceability per CONTEXT.

9. Return NextResponse.json({ received: true }, { status: 200 }).

Note: PaymentLog outcome values: 'paid' | 'canceled' | 'ignored'. Map payload.status 'paid' → 'paid', 'failed'|'expired' → 'canceled'.

For next.config.ts: if body size limit is needed for webhooks, add experimental.serverActions.bodySizeLimit or api.bodyParser = false equivalent. Next.js 15 route handlers receive raw request; no config change needed unless body is very large.

Create src/app/api/payments/create/route.ts:

1. POST handler. Auth: const session = await getSessionFromCookies(); if (!session) return NextResponse.json({ error: 'Não autenticado' }, { status: 401 }).

2. Parse body: { orderId: string, method: 'pix' | 'crypto' }. Validate orderId and method present; method must be 'pix' or 'crypto'.

3. Load order: const order = await prisma.order.findUnique({ where: { id: orderId }, include: { plan: true } }). If !order return 404. If order.userId !== session.user.id return 403. If order.status !== 'pending_payment' return 400 { error: 'Pedido não está aguardando pagamento' }.

4. PLAN-03: const hasActive = await hasActiveOrder(session.user.id); if (hasActive) return NextResponse.json({ error: 'Você já possui uma máquina ativa. Aguarde o término para contratar outra.' }, { status: 409 }).

5. Get amount: order.amountCents ?? order.plan.priceCents. If missing, return 400.

6. Gateway: const gateway = createMockGateway(); const result = await gateway.createPaymentIntent({ orderId, amountCents, currency: order.currency ?? 'BRL', method, metadata: { userId: session.user.id } }).

7. Return 201: NextResponse.json({ paymentId: result.paymentId, qrCode: result.qrCode, redirectUrl: result.redirectUrl }, { status: 201 }).
