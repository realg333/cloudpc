<objective>
Add session diagnostics and improve email error handling. Enables root-cause identification and user feedback when email fails.
</objective>

<tasks>

### Task 1: Session diagnostic logging
- In `getSessionFromCookies()`, when `process.env.SESSION_DEBUG === 'true'`:
  - Log whether cookie is present (cookieStore.get(SESSION_COOKIE) exists)
  - Log JWT verify result (before DB lookup): valid payload or error
  - Log DB lookup result: session found, expired, or not found
- Use `console.warn` with prefix `[session-debug]` so logs are easy to filter
- Do NOT log token value or session ID (security)

### Task 2: Email return value and error logging
- Change `sendVerificationEmail` to return `Promise<boolean>`: true if sent, false if failed or RESEND_API_KEY missing
- When Resend returns `error`, log: `console.error('[email] Resend error:', JSON.stringify(error))`
- When `resend` is null (no API key), return false (already logs warning)

### Task 3: Signup redirect with email status
- In signup route, await `sendVerificationEmail` and capture result
- Redirect to `/login?signup=1&email_sent=1` if email sent, else `/login?signup=1&email_failed=1`
- Login page already receives signup param; add handling for email_failed to show "Verifique sua caixa de entrada. Se não receber, solicite reenvio na tela de login."

### Task 4: .env.example Vercel checklist
- Add section "Vercel deployment checklist" with:
  - SESSION_SECRET (required; generate with openssl rand -base64 32)
  - RESEND_API_KEY (required for verification emails)
  - RESEND_FROM_EMAIL (optional; use verified domain for production; onboarding@resend.dev is test-only)
  - SESSION_DEBUG (optional; set to 'true' to log session diagnostics; remove in production)
- Add note: "Verify domain at https://resend.com/domains for production emails"

</tasks>
